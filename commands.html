;agree - Agree to terms of the server, if it is required by the server staff.
;agreement - Require users to agree to terms before they get full access to the server. (Guild-only command)
Usage: ;agreement [setting] [value]


Gives users a temporary role on join that you should configure to hide all channels except the rules/TOS.
The channel you set should be where your rules/TOS are posted. Bender will delete all messages in that channel that are not pinned and posted less than 2 weeks ago.
To accept the agreement, users must use the agree command in the above channel or in DM.
The message is what the bot will DM users upon join. If a user does not agree within an hour, they will be sent a reminder to do so, then kicked after another hour if they don't follow through.
If the user has DMs disabled, Bender will mention them in the agreement channel and delete the message after 30 seconds.
The agreement system may be enabled or disabled at any time.
You can also change the wait time (in minutes or hours) that a user must be a member before agreeing.
Default permission: MANAGE_GUILD

Examples:
;agreement channel #tos
;agreement role n00b
;agreement message Welcome to our server! Please read #rules and #tos before typing, and enjoy your stay!
;agreement enable

;tag - Add, edit, view, or delete a tag ("custom command"), or list all tags, for the server. (Guild-only command)
Usage: ;tag <name | "list"> [content | "delete"]


Use "quotation marks" to use spaces in tag names, e.g. "rule #4".
You can use {mention} to echo the first mentioned user in the message using the tag, if applicable.
Or, use {mention.tag} for their Discord tag (no mention) or {mention.id} for their ID.
The variable {args} can be used to repeat the user's message...
For example, creating a tag with tag gay {args} is gay then typing gay Joe would produce Joe is gay.
The variable {author} can be used to repeat the person who sent the tag message.
The variables {author.tag} and {author.id} work the same way as the mention variables above.
For example, if a tag is made with tag banme {author} has been banned. and fred uses the tag, it will say @fred has been banned.
{p} can be used to show the current command prefix, i.e. {p}ban might produce ;ban and then !ban after changing the prefix.
Default permission: MANAGE_MESSAGES

Examples:
;tag ayy lmao
;tag "demote me" You have been demoted. üëã
;tag bam delete

;purge - Deletes a specified number of messages in the channel. (Guild-only command)
Usage: ;purge <# of messages> [filter]


Number of messages must be between 2 and 1000 (inclusive.)
The filter searches within the range of messages. (e.g. purge 1000 images will purge all images within the last 1000 messages.)
Due to a limitation of Discord, messages older than 14 days will not be deleted.

All available filters
invites - Discord invite links
@user | user ID - Sent by user | me - Purge only your own messages
match regex | not regex - Content matches/doesn't match regex.
bots** | humans - Sent by bot/non-bot users
images - Contains images as an attachment or embed
embeds - Contains a "rich" embed (not link embeds ~ rich embeds can only be generated by bots and selfbots)
emotes - Contains custom emotes (uploaded to Discord, not Unicode emoji)
before: <id> | after: <id> - Only searches for messages before/after the one specified. [Cannot be used together!]
pins - include pinned messages (excluded by default.)

NOTICE! Filters can be mixed and matched, so be careful not to include any extra text in the command, as it might trigger other filters.
* regex = Regular expression. Special character reference can be found on MDN.
To select messages that start with text, use match ^text, and for ending with text, use match text$.*
Default permission: MANAGE_MESSAGES

Aliases: ;prune, ;del
Examples:
;purge 20
;purge 100 links
;prune 1000 images
;del 50 @Joe

;automod - Change automod settings. (Guild-only command)
Usage: ;automod [setting] [value]


Available settings
enable/disable - Self-explanatory; enables or disables automod entirely.
no-invite true|false - Set to true/enable to disallow invites being posted.
selfbot - true|false - Whether to allow selfbots. Disallowed by default, set to true to allow them.
mentions - Maximum number of mentions in a message. More than this will result in the message deleted and a warning. 0 to disable.
mention-ban - Maximum number of mentions in a message before automatically banning the user. Default is 20. 0 to disable.
spam - Configure quick message spam protection, with the format # messages/time. More details in examples. Default is 6 messages/5 seconds.
mute - Configure automute, with the format # violations/time. More details in examples. Default is 3 violations/1 min.

NOTE: ignore and ignore-ch have been moved to the ignore command.
Default permission: MANAGE_CHANNELS

Examples:
;automod no-invite enable
;automod mentions 10
;automod mute 4/30sec
;automod spam 5/3sec
;automod spam 3/800ms

Before each of these messages, there will be a notice such as "You have been kicked/muted/banned from {server}." to avoid the recipient being confused.

;perms - Sets who has permissions to use which commands. (Guild-only command)
Usage: ;perms <command/group name | "use_tags"> [role | discord permission | "default"]


"Discord permission" refers to an item on this list. It's generally the same names as the UI, with underscores instead of spaces.
Any permissions set here will override the defaults; individual command permissions override group permissions as well.
You can use @everyone or everyone to make a command available to all members.
You can also use "reset" to reset the permissions.
To view the permissions for a command, just omit the second argument.
Append + to the end of a role name to make the perms hierarchal (the __ role and above.) Otherwise it will be limited to members that explicitly have the specified role.
use_tags is a special permission determining who can use the "custom commands" created by the tag command.
Default permission: ADMINISTRATOR

Aliases: ;permissions
Examples:
;perms pong shitposter
;perms ping Moderator+
;perms mod ban_members

;welcome - Configure the messages that can be sent when users join and leave the server. (Guild-only command)
Usage: ;welcome <setting> [value]


Available settings:
Setting any of the settings to "none" will delete the value instead of using the text "none".
channel - The channel in which the following messages are sent:
message - The message to be posted when a user joins the server.
dm - A message to be sent to the user's PM when they join the server. note: if they have DMs enabled it will mention them in the welcome channel then delete the message after 30 seconds
leave-msg - Message to be posted when a user leaves the server.
ban-msg - When a user is banned, post this, blah blah blah...

Values that can be used in join/leave/ban messages:
{id} - The member's ID. {user} - Shows a user's "tag", e.g. Joe üé∏#7070.
{member} - Mentions the server member (recommended only for the join message, as ban/leave mentions will turn into ID's shortly after the user is gone.)
{server} - The name of the server. {count} - The number of server members.
Values only available in join messages:
{today} - Number of members gained today. {thisweek} Number of members gained this week.
{new} - Shows when the user's account was created, if it was less than 2 days ago.
Note: adding th to the end of {count}, {today}, or {thisweek} will properly format the number; For example, {count}th member would produce "2nd member", "23rd member", "55th member" etc.
Default permission: MANAGE_GUILD

Examples:

;welcome ban-msg {user} got their dumb ass banned!
